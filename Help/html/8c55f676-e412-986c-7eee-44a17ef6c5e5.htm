count of available top-level views e.g. screens in a multi screen setting

Does not dispose this singleton because it cant be destroyed! But it disposes the related Adapter manager and so on if they are disposable.

Recalibrate
Forces the current active adapter devices to recalibrate.
RecalibrateAll
Forces all connected adapter devices to recalibrate.
RefreshDisplay
Tries to sent the actual build matrix to all devices, that are active. To enable a sending, the pins have to be unlocked (still rendering or maybe locked by the user) and at an Adapter has to be active.
RemoveView
remove view
RenameView
rename a view
RenderDisplay
Forces the rendering thread to build the resulting Matrix by calling all renderer for the visible view ranges. The matrix will not been sent until the refresh timer is elapsed or the RefreshDisplay(Boolean) Method was called.
ShowView
show a view. will be displayed with all other visible views at next display update.
ToString
Returns a string that represents the current object.
(Inherited from Object.)
UnlockPins
Unlocks the pins. Enables renderers to refresh the matrix that is send to the devices
